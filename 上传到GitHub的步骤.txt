🚀 上传 LibreTranslate 项目到 GitHub 的详细步骤
==============================================

方法 1：使用 GitHub Desktop（最简单，推荐） ⭐
----------------------------------------

1. 打开 GitHub Desktop 应用

2. 点击菜单 File → Add Local Repository
   （或使用快捷键 Cmd+O）

3. 在弹出的窗口中：
   - 点击 "Choose..." 按钮
   - 导航到：/Users/enithz/Desktop/translation
   - 选择该文件夹
   - 点击 "Open"

4. 如果提示 "This directory does not appear to be a Git repository"：
   - 点击 "create a repository" 链接
   - 或者点击 "Create Repository" 按钮

5. 在 "Create a new repository" 窗口中：
   - Repository name: translation
   - Description: LibreTranslate cloud service for Musa browser
   - 不要勾选 "Initialize this repository with a README"（我们已经有了）
   - 点击 "Create Repository"

6. 在主界面左下角：
   - Summary 输入：Initial commit: LibreTranslate cloud service
   - Description 输入：Ready for Zeabur deployment
   - 点击 "Commit to main" 按钮

7. 点击顶部的 "Publish repository" 按钮

8. 在弹出窗口中：
   - Name: translation
   - Description: LibreTranslate cloud service for Musa browser
   - 取消勾选 "Keep this code private"（让仓库公开，这样 Zeabur 才能访问）
   - 点击 "Publish Repository"

9. 等待上传完成，会看到 "Published repository" 的提示

10. 验证：访问 https://github.com/zt6453928/translation
    应该能看到所有文件


方法 2：使用终端命令行
----------------------

打开"终端"应用（Terminal.app），复制粘贴以下命令：

# 1. 进入项目目录
cd /Users/enithz/Desktop/translation

# 2. 初始化 Git 仓库
git init

# 3. 添加所有文件
git add .

# 4. 提交更改
git commit -m "Initial commit: LibreTranslate cloud service for Musa browser"

# 5. 添加远程仓库
git remote add origin https://github.com/zt6453928/translation.git

# 6. 创建并切换到 main 分支
git branch -M main

# 7. 推送到 GitHub
git push -u origin main


⚠️ 重要提示：
当要求输入用户名和密码时：
- Username: 输入你的 GitHub 用户名（zt6453928）
- Password: 不要输入你的 GitHub 密码！而是使用 Personal Access Token

如何获取 Personal Access Token：
1. 访问：https://github.com/settings/tokens
2. 点击 "Generate new token" → "Generate new token (classic)"
3. Note: 输入 "LibreTranslate Upload"
4. Expiration: 选择 "90 days" 或 "No expiration"
5. 勾选权限：
   ☑️ repo（全部子选项）
6. 点击 "Generate token"
7. 复制生成的 token（只显示一次，请保存好）
8. 在终端密码提示处粘贴这个 token


方法 3：使用 GitHub 网页上传（适合小项目）
----------------------------------------

1. 访问：https://github.com/zt6453928/translation

2. 如果仓库是空的：
   - 点击 "uploading an existing file" 链接
   
3. 将以下文件拖拽到网页中：
   - 打开 Finder
   - 导航到 /Users/enithz/Desktop/translation
   - 全选所有文件（Cmd+A）
   - 拖拽到 GitHub 网页的上传区域

4. 填写 commit 信息：
   - Commit changes 输入：Initial commit: LibreTranslate service

5. 点击 "Commit changes" 按钮

下一步：部署到 Zeabur
--------------------

上传成功后：

1. 访问：https://zeabur.com

2. 点击 "New Project"

3. 选择 "Deploy from GitHub"

4. 选择 "zt6453928/translation" 仓库

5. Zeabur 会自动检测并部署

6. 等待 3-5 分钟完成部署

7. 获取服务 URL（类似：https://translation-xxxx.zeabur.app）

8. 测试服务：
   curl https://translation-xxxx.zeabur.app/languages

9. 更新 Musa 配置：
   编辑 /Users/enithz/Desktop/Musa/Musa/TranslationManager.swift
   修改 translateServiceURL 为你的 Zeabur URL


如果遇到问题
-----------

1. GitHub Desktop 问题：
   - 重启 GitHub Desktop
   - 检查是否登录了正确的账号

2. 终端推送失败：
   - 确保使用的是 Personal Access Token 而不是密码
   - 检查 token 权限是否包含 repo

3. 仓库不存在：
   - 访问 https://github.com/new
   - 创建名为 "translation" 的新仓库
   - 保持为空仓库（不要添加 README）
   - 然后重新执行上传步骤

4. 其他问题：
   - 查看 UPLOAD_INSTRUCTIONS.md
   - 查看 DEPLOY.md


需要帮助？
---------

如果以上方法都无法完成上传，可以：
1. 检查 GitHub 账号是否正常
2. 检查网络连接
3. 尝试重启电脑
4. 联系 GitHub 支持
